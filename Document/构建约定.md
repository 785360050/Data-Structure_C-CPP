# 构建约定

当前仓库在打通 `Data_Sturcture/C++` 的 CMake 时，先遵守以下约定：

1. 默认构建目录使用 `Build`
   - 约定命令：`cmake -S . -B Build`

2. 可执行文件按用途分目录，不在文件名中重复表达用途
   - 示例程序统一输出到 `Build/bin/Demo/`
   - 单元测试统一输出到 `Build/bin/UnitTest/`

3. 输出文件名只保留模块名
   - 正确示例：`Build/bin/Demo/Sequential_List`
   - 正确示例：`Build/bin/UnitTest/Sequential_List`
   - 不接受：`Build/bin/Demo/Demo_Sequential_List`
   - 不接受：`Build/bin/UnitTest_Sequential_List`

4. 现阶段只处理 `Data_Sturcture/C++`
   - `Data_Sturcture/C`
   - `Data_Sturcture/Library_C++`
   - `Data_Sturcture/Experimental`
   - `QT`
   以上部分暂不纳入当前主线构建整理范围

补充说明：

- `Data_Sturcture/C` 的定位为“参考实现”
- 后续要求它保持可配置、可构建、可运行
- 但不作为当前主线持续演进对象
- 其构建建议使用独立开关控制，避免影响 `C++` 主线整理

5. 先打通最小完整样板，再横向复制
   - 样板要求同时包含源码示例和单元测试
   - 最初样板模块为 `Sequential_List`
   - 当前已经从样板阶段扩展到一批基础模块
