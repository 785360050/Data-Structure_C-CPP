# Tree_Binary_Search ADT 定义

## 1. 定义对象

`Tree_Binary_Search` 表示一个按关键字有序组织的二叉树。

本 ADT 关心的是关键字查找、插入、删除，以及二叉搜索树的不变式。

## 2. 公共状态语义

对于任意一个 `Tree_Binary_Search` 对象，ADT 层只关心以下逻辑状态：

- 当前是否存在根结点
- 当前包含哪些关键字
- 每个关键字对应哪个元素
- 当前结点个数 `count`
- 整棵树是否满足二叉搜索树的有序关系

本 ADT 不关心底层采用普通二叉搜索树、AVL 树还是红黑树实现。

## 3. 关键字约定

本项目中的 `Tree_Binary_Search` 采用以下公共约定：

- 每个结点由 `key` 与对应元素共同确定
- 关键字在同一棵树中应唯一
- 对任意结点，其左子树中的关键字都小于该结点关键字
- 对任意结点，其右子树中的关键字都大于该结点关键字

## 4. 公共操作与契约

本节采用按需描述的接口说明风格。

只写该操作真正需要的项目，不强制为每个操作补齐“无”的字段。

### 4.1 插入元素 `Element_Insert(key, element)`
语义：
- 将关键字为 `key`、元素值为 `element` 的结点插入当前二叉搜索树
参数：
- `key`：待插入关键字
- `element`：待插入元素值
约束：
- `key` 在当前树中应唯一
效果：
- 插入后，关键字 `key` 属于当前树
- 插入后，整棵树仍满足二叉搜索树有序关系
错误语义：
- 若 `key` 已存在，应抛出异常或以统一错误方式拒绝插入

### 4.2 删除元素 `Element_Delete(key)`
语义：
- 删除关键字为 `key` 的结点
参数：
- `key`：待删除关键字
约束：
- `key` 应存在于当前树中
效果：
- 删除后，关键字 `key` 不再属于当前树
- 删除后，整棵树仍满足二叉搜索树有序关系
错误语义：
- 若 `key` 不存在，应抛出异常或以统一错误方式拒绝删除

### 4.3 查找结点 `Node_Search(key)`
语义：
- 查找关键字为 `key` 的结点
参数：
- `key`：待查找关键字
返回值：
- 若查找成功，返回对应结点
- 若查找失败，返回空指针或等价空值
效果：
- 不改变树逻辑状态

### 4.4 前序遍历 `Traverse_PreOrder(node)`
语义：
- 以前序方式遍历以 `node` 为根的当前子树
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

### 4.5 中序遍历 `Traverse_InOrder(node)`
语义：
- 以中序方式遍历以 `node` 为根的当前子树
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

### 4.6 后序遍历 `Traverse_PostOrder(node)`
语义：
- 以后序方式遍历以 `node` 为根的当前子树
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

### 4.7 层次遍历 `Traverse_LevelOrder(node)`
语义：
- 以层次顺序遍历以 `node` 为根的当前子树
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

### 4.8 `Get_Depth(node)`
语义：
- 返回以 `node` 为根的当前子树深度
参数：
- `node`：深度计算起点
返回值：
- 返回当前子树的层数
效果：
- 不改变树逻辑状态

## 5. 一致性要求

ADT 只约束“行为结果”，不约束“内部实现方式”。
任意的 `Tree_Binary_Search` 实现，只要都声明自己满足本 ADT，就应共享以下行为：

- 相同的关键字唯一性语义
- 相同的二叉搜索树有序关系
- 相同的查找结果语义
- 相同的插入和删除后有序性保持语义
- 相同的遍历顺序语义

实现之间可以存在差异，但这些差异不得改变上述公共行为。
