# Tree ADT 定义

## 1. 定义对象

`Tree` 表示一个具有层次关系的有限结点集合。

本项目中的 `Tree` 以根结点为起点，采用孩子兄弟表示法描述一般树结构。

## 2. 公共状态语义

对于任意一个 `Tree` 对象，ADT 层只关心以下逻辑状态：

- 当前是否存在根结点
- 结点之间的父子关系
- 同层兄弟结点之间的相对次序
- 当前结点个数 `count`

本 ADT 不关心结点对象如何分配，也不关心具体结点类型如何存放附加数据。

## 3. 结构约定

本项目中的 `Tree` 采用孩子兄弟表示法。

约定如下：

- `left` 语义表示某结点的第一个孩子
- `right` 语义表示某结点的下一个兄弟
- 根结点没有双亲
- 遍历操作针对的是以指定结点为根的当前子树

## 4. 公共操作与契约

本节采用按需描述的接口说明风格。

只写该操作真正需要的项目，不强制为每个操作补齐“无”的字段。

### 4.1 `Tree_CheckEmpty`
语义：
- 判断当前树是否为空树
返回值：
- 当且仅当树中不存在任何结点时返回 `true`
效果：
- 不改变树逻辑状态

### 4.2 `Tree_GetRoot`
语义：
- 返回当前树的根结点
返回值：
- 若树非空，返回根结点
- 若树为空，返回空指针或等价空值
效果：
- 不改变树逻辑状态

### 4.3 插入结点 `Node_Insert(node, parent, pos)`
语义：
- 将结点 `node` 连接到 `parent` 的指定逻辑位置
参数：
- `node`：待插入结点
- `parent`：双亲结点
- `pos`：插入位置
约束：
- `parent` 应存在于当前树中
- `pos` 应符合孩子兄弟表示法约定
效果：
- 当 `pos` 表示孩子位置时，`node` 成为 `parent` 的第一个孩子
- 当 `pos` 表示兄弟位置时，`node` 成为 `parent` 的下一个兄弟
- 树的结点个数增加一
错误语义：
- 若 `parent` 不存在或 `pos` 非法，应抛出异常或以统一错误方式拒绝插入

### 4.4 前序遍历 `Tree_Traverse_PreOrder(node)`
语义：
- 以前序方式遍历以 `node` 为根的当前子树
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

### 4.5 中序遍历 `Tree_Traverse_InOrder(node)`
语义：
- 以当前项目约定的孩子兄弟表示结构执行中序遍历
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

### 4.6 后序遍历 `Tree_Traverse_PostOrder(node)`
语义：
- 以后序方式遍历以 `node` 为根的当前子树
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

### 4.7 层次遍历 `Tree_Traverse_LevelOrder(node)`
语义：
- 以层次顺序遍历以 `node` 为根的当前子树
参数：
- `node`：遍历起点
效果：
- 不改变树逻辑状态

## 5. 一致性要求

ADT 只约束“行为结果”，不约束“内部实现方式”。
任意的 `Tree` 实现，只要都声明自己满足本 ADT，就应共享以下行为：

- 相同的根结点语义
- 相同的孩子兄弟表示语义
- 相同的插入位置语义
- 相同的遍历起点语义
- 相同的空树判定语义

实现之间可以存在差异，但这些差异不得改变上述公共行为。
