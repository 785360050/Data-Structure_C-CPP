# Stack ADT 定义

## 1. 定义对象

`Stack` 表示一个仅允许在同一端进行插入和删除的有限元素序列。

本 ADT 采用后进先出语义，即最后压入栈的元素最先出栈。

## 2. 公共状态语义

对于任意一个 `Stack` 对象，ADT 层只关心以下逻辑状态：

- 元素的先后顺序
- 当前元素个数 `size`
- 当前是否存在栈顶元素

本 ADT 不关心底层元素如何存放，也不关心是否存在额外预留空间或容量上限。

## 3. 栈顶约定

本项目中的 `Stack` 采用“栈顶”作为唯一公共操作端。

约定如下：

- 栈顶元素是最近一次入栈且尚未出栈的元素
- 当 `size == 0` 时，栈中不存在栈顶元素
- 读取、插入、删除都只对栈顶端发生作用

## 4. 公共操作与契约

本节采用按需描述的接口说明风格。

只写该操作真正需要的项目，不强制为每个操作补齐“无”的字段。

### 4.1 `Is_Empty`
语义：
- 判断栈当前是否为空
返回值：
- 当且仅当 `size == 0` 时返回 `true`
效果：
- 不改变栈逻辑状态

### 4.2 `Get_Size`
语义：
- 返回当前栈中元素个数
返回值：
- 返回值应等于当前逻辑元素数量
效果：
- 不改变栈逻辑状态

### 4.3 `Get_Top`
语义：
- 读取当前栈顶元素
约束：
- 栈非空
返回值：
- 当前栈顶元素的引用
效果：
- 栈逻辑状态不发生变化
错误语义：
- 若栈为空，应抛出异常

### 4.4 入栈 `Element_Push(element)`
语义：
- 将新元素压入栈顶
参数：
- `element`：待入栈元素
效果：
- 新栈长度为原长度加一
- 新元素成为新的栈顶元素
- 原有元素的相对顺序保持不变

### 4.5 出栈 `Element_Pop`
语义：
- 删除当前栈顶元素
约束：
- 栈非空
效果：
- 新栈长度为原长度减一
- 原次栈顶之下的元素成为新的栈顶元素
- 其余元素的相对顺序保持不变
错误语义：
- 若栈为空，应抛出异常

### 4.6 `Clear`
语义：
- 将栈重置为空栈
效果：
- `Get_Size() == 0`
- `Is_Empty() == true`

## 5. 一致性要求

ADT 只约束“行为结果”，不约束“内部实现方式”。
任意的 `Stack` 实现，只要都声明自己满足本 ADT，就应共享以下行为：

- 相同的栈顶定义
- 相同的后进先出语义
- 相同的空栈判定语义
- 相同的清空后逻辑状态
- 空栈读取栈顶和空栈出栈时的错误语义类别应一致

实现之间可以存在差异，但这些差异不得改变上述公共行为。
