# String ADT 定义

## 1. 定义对象

`String` 表示一个按顺序排列的有限字符序列。

本 ADT 关心的是字符内容、字符顺序以及子串匹配语义。

## 2. 公共状态语义

对于任意一个 `String` 对象，ADT 层只关心以下逻辑状态：

- 字符的先后顺序
- 当前字符个数 `length`
- 指定位置上是否存在字符

本 ADT 不关心底层字符数组如何分配，也不关心匹配算法的具体实现方式。

## 3. 位置约定

本项目中的 `String` 位置参数采用字符串下标语义。

约定如下：

- 首字符位置为 `0`
- 末字符位置为 `length - 1`
- 合法读取位置范围为 `[0, length - 1]`
- 子串匹配结果返回的是首个匹配字符的位置

## 4. 公共操作与契约

本节采用按需描述的接口说明风格。

只写该操作真正需要的项目，不强制为每个操作补齐“无”的字段。

### 4.1 读取指定位置字符 `operator[](index)`
语义：
- 读取位置 `index` 上的字符
参数：
- `index`：待读取字符的位置
约束：
- `index` 属于 `[0, length - 1]`
返回值：
- 位置 `index` 对应字符的引用
效果：
- 字符串逻辑状态不发生变化
错误语义：
- 若 `index` 不在合法范围内，应抛出异常

### 4.2 朴素匹配 `String_Fetch_Brute_Force(substring)`
语义：
- 在当前字符串中查找子串 `substring` 的首次出现位置
参数：
- `substring`：待匹配子串
返回值：
- 若匹配成功，返回首个匹配位置
- 若匹配失败，返回 `-1`
- 若 `substring` 为空串，返回 `0`
效果：
- 字符串逻辑状态不发生变化

### 4.3 KMP 匹配 `String_Fetch_KMP(substring)`
语义：
- 在当前字符串中查找子串 `substring` 的首次出现位置
参数：
- `substring`：待匹配子串
返回值：
- 若匹配成功，返回首个匹配位置
- 若匹配失败，返回 `-1`
- 若 `substring` 为空串，返回 `0`
效果：
- 字符串逻辑状态不发生变化

## 5. 一致性要求

ADT 只约束“行为结果”，不约束“内部实现方式”。
任意的 `String` 实现，只要都声明自己满足本 ADT，就应共享以下行为：

- 相同的字符顺序语义
- 相同的位置编号约定
- 相同的首次匹配语义
- 相同的空串匹配语义
- 相同的未匹配返回语义

实现之间可以存在差异，但这些差异不得改变上述公共行为。
