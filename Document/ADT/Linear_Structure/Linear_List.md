# Linear_List ADT 定义

## 1. 定义对象

`Linear_List` 表示一个按线性顺序排列的有限元素序列。

序列中的每个元素都对应一个确定的位置，除首元素外每个元素都有唯一直接前驱，除尾元素外每个元素都有唯一直接后继。

本项目中的 `Linear_List` 采用教材式“位序”表示位置。

## 2. 公共状态语义

对于任意一个 `Linear_List` 对象，ADT 层只关心以下逻辑状态：

- 元素的先后顺序
- 当前元素个数 `size`
- 指定位置上是否存在元素

本 ADT 不关心底层元素如何存放，也不关心是否存在额外预留空间。

## 3. 位置约定

本项目对 `Linear_List` 的位置参数使用“位序”语义，而不是数组下标语义。

约定如下：

- 首元素位序为 `1`
- 末元素位序为 `size`
- 合法插入位置范围为 `[1, size + 1]`
- 合法读取、更新、删除位置范围为 `[1, size]`

说明：

- 这里的位序是 ADT 的逻辑位置，不要求对应任何实际物理下标
- 即使底层实现是数组，ADT 层也仍然只讨论位序

## 4. 公共操作与契约

本节采用按需描述的接口说明风格。

只写该操作真正需要的项目，不强制为每个操作补齐“无”的字段。

### 4.1 `Is_Empty`
语义：
- 判断线性表当前是否为空
返回值：
- 当且仅当 `size == 0` 时返回 `true`
效果：
- 不改变线性表逻辑状态

### 4.2 `Get_Size`
语义：
- 返回当前线性表中元素个数
返回值：
- 返回值应等于当前逻辑元素数量
效果：
- 不改变线性表逻辑状态

### 4.3 读取指定位置元素 `operator[](pos)`
语义：
- 读取位序为 `pos` 的元素
参数：
- `pos`：待读取元素的位序
约束：
- `pos` 属于 `[1, size]`
返回值：
- 位序 `pos` 对应元素的引用
效果：
- 线性表逻辑状态不发生变化
错误语义：
- 若 `pos` 不在合法范围内，应抛出异常

### 4.4 插入元素 `Element_Insert(pos, elem)`
语义：
- 在位序 `pos` 处插入新元素
- 原位于 `pos` 及之后的元素整体后移一个逻辑位置
参数：
- `pos`：插入位置的位序
- `elem`：待插入元素
约束：
- `pos` 属于 `[1, size + 1]`
效果：
- 新表长度为原长度加一
- 新元素成为位序 `pos` 的元素
- 原位序区间 `[pos, size]` 的元素顺序保持不变，但位序整体加一
错误语义：
- 若 `pos` 不在合法范围内，应抛出异常

### 4.5 删除元素 `Element_Delete(pos)`
语义：
- 删除位序 `pos` 的元素
- 原位于 `pos + 1` 及之后的元素整体前移一个逻辑位置
参数：
- `pos`：待删除元素的位序
约束：
- `pos` 属于 `[1, size]`
效果：
- 新表长度为原长度减一
- 原位序区间 `[pos + 1, size]` 的元素顺序保持不变，但位序整体减一
错误语义：
- 若 `pos` 不在合法范围内，应抛出异常

### 4.6 更新元素 `Element_Update(pos, elem)`
语义：
- 将位序 `pos` 上的元素值替换为 `elem`
参数：
- `pos`：待更新元素的位序
- `elem`：新元素值
约束：
- `pos` 属于 `[1, size]`
效果：
- 表长度不变
- 只有位序 `pos` 上的元素值被替换
- 其他元素的相对顺序和取值不变
错误语义：
- 若 `pos` 不在合法范围内，应抛出异常

### 4.7 `List_Clear`
语义：
- 将线性表重置为空表
效果：
- `Get_Size() == 0`
- `Is_Empty() == true`

## 5. 一致性要求

ADT 只约束“行为结果”，不约束“内部实现方式”。
任意的 `Linear_List` 实现，只要都声明自己满足本 ADT，就应共享以下行为：

- 相同的位序定义
- 相同的合法位置范围
- 相同的插入、删除、更新逻辑语义
- 相同的空表判定语义
- 相同的清空后逻辑状态
- 非法位序访问时的错误语义类别应一致

实现之间可以存在差异，但这些差异不得改变上述公共行为。
