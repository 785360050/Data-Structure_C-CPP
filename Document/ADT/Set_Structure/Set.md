# Set ADT 定义

## 1. 定义对象

`Set` 表示一个由互不重复元素组成的有限集合。

本 ADT 关心的是元素是否属于集合，以及集合之间的关系与运算结果。

## 2. 公共状态语义

对于任意一个 `Set` 对象，ADT 层只关心以下逻辑状态：

- 当前包含哪些元素
- 当前元素个数
- 任意两个元素是否被视为相同元素

本 ADT 不关心底层元素如何存放，也不关心使用哈希、顺序存储还是其他实现方式。

## 3. 元素约定

本项目中的 `Set` 采用集合语义，而不是线性序列语义。

约定如下：

- 同一元素在集合中至多出现一次
- 集合不关心元素排列顺序
- 元素是否相同由实现选定的等价判定规则决定，并且在同一 ADT 下应保持一致

## 4. 公共操作与契约

本节采用按需描述的接口说明风格。

只写该操作真正需要的项目，不强制为每个操作补齐“无”的字段。

### 4.1 插入元素 `Insert(element)`
语义：
- 将 `element` 加入当前集合
参数：
- `element`：待插入元素
效果：
- 若 `element` 原本不属于集合，则插入后属于集合
- 若 `element` 原本已属于集合，则集合逻辑状态不应重复增加同一元素

### 4.2 删除元素 `Delete(element)`
语义：
- 将 `element` 从当前集合中移除
参数：
- `element`：待删除元素
效果：
- 若 `element` 原本属于集合，则删除后不再属于集合
- 若 `element` 原本不属于集合，则集合逻辑状态保持不变

### 4.3 成员判断 `Has(element)`
语义：
- 判断 `element` 是否属于当前集合
参数：
- `element`：待判断元素
返回值：
- 当且仅当 `element` 属于当前集合时返回 `true`
效果：
- 不改变集合逻辑状态

### 4.4 子集判断 `Has(subset)`
语义：
- 判断参数集合 `subset` 是否为当前集合的子集
参数：
- `subset`：待判断集合
返回值：
- 当且仅当 `subset` 中每个元素都属于当前集合时返回 `true`
效果：
- 不改变集合逻辑状态

### 4.5 `Size`
语义：
- 返回当前集合中的元素个数
返回值：
- 返回值应等于当前逻辑元素数量
效果：
- 不改变集合逻辑状态

### 4.6 `Is_Empty`
语义：
- 判断当前集合是否为空
返回值：
- 当且仅当集合中不含任何元素时返回 `true`
效果：
- 不改变集合逻辑状态

### 4.7 `Clear`
语义：
- 将当前集合重置为空集合
效果：
- 集合中不再包含任何元素

### 4.8 并集运算 `Union(set)`
语义：
- 令当前集合更新为其与参数集合 `set` 的并集
参数：
- `set`：参与运算的集合
效果：
- 运算后，当前集合包含原集合与参数集合中所有出现过的元素

### 4.9 交集运算 `Intersection(set)`
语义：
- 令当前集合更新为其与参数集合 `set` 的交集
参数：
- `set`：参与运算的集合
效果：
- 运算后，当前集合只保留同时属于两集合的元素

### 4.10 差集运算 `Difference(set)`
语义：
- 令当前集合更新为当前集合与参数集合 `set` 的差集
参数：
- `set`：参与运算的集合
效果：
- 运算后，当前集合只保留原本属于当前集合但不属于参数集合的元素

## 5. 一致性要求

ADT 只约束“行为结果”，不约束“内部实现方式”。
任意的 `Set` 实现，只要都声明自己满足本 ADT，就应共享以下行为：

- 相同的元素唯一性语义
- 相同的成员判断语义
- 相同的子集判断语义
- 相同的并、交、差运算结果语义
- 相同的空集合判定语义

实现之间可以存在差异，但这些差异不得改变上述公共行为。
